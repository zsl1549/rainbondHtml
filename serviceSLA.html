好雨Rainbond Cloud服务等级协议（SLA）

  1. 术语和定义

    1.1   Rainbond Cloud 服务 好雨为您提供的 Rainbond Cloud 服务，包括DevOps、Srevisemesh微服务、CI/CD、自动化运维、无侵入架构、多云管理、中台制品库等功能服务，具体以您购买的服务以及 Rainbond Cloud 提供的服务内容为准。您可以通过本服务使技术团队在云端高效协同，实践 DevOps，打通应用开发、测试、交付、运维完整链条，提升软件交付质量与速度。

    1.2   服务不可用 Rainbond Cloud 服务不可用单位时间：Rainbond Cloud 服务在单位时间内（5 分钟为一个统计粒度）错误率大于 0.05%（不含本数），视为该单位时间服务不可用。在连续 10 分钟或更长时间内， DevOps 服务均持续不可用才计入服务不可用时间，低于连续 10 分钟不计入服务不可用时间。 错误率： 错误率是指由 Rainbond Cloud 返回的失败请求数占单位时间内（5分钟内）用户请求数的比例。 失败请求： 指 Rainbond Cloud 返回的错误码为服务器内部错误码的请求，包括 Internal Error（500错误） 或 Service Unavailable（503错误）两种错误码；但不包括触发频控导致的限流请求或者 Rainbond Cloud 升级、变更、停机而导致的失败请求。用户请求指的是 Rainbond Cloud 服务器端接收到的用户发送的请求，但不包括未经身份验证、鉴权失败或者欠费停服状态下的请求。用户端由于黑客攻击而对 Rainbond Cloud 的请求，或者由于配置了跨区域复制、生命周期规则而在后端异步执行的请求，均不视为有效请求或失败请求。

    1.3   服务不可用时间合计 服务不可用时间为服务月度内 Rainbond Cloud 服务不可用的分钟数之和。

    1.4   服务月度 服务月度是指您购买本服务的服务期限所包含的各个自然月度，如您购买三个月本服务，服务开通之日为 03 月 17 日，则包含 4 个服务月度，其中第 1 个服务月度是指 03 月 17 日到 03 月 31 日，第 2 个服务月度是指 04 月 01 日到 04 月 30 日，第 3 个服务月度是指 05 月 01 日到 05 月 31 日，第 4 个服务月度是指 06 月 01 日到 06 月16 日。服务可用性按服务月度单独核算。

    1.5   月度服务费用 Rainbond Cloud 服务为预付费模式，月度服务费用指您在一个服务月度中就 Rainbond Cloud 服务所实际消费的服务费用总额，不包含已经支付尚未消费的部分。 月度服务费不含用代金券、优惠券、服务费减免等抵扣的费用。

  2. 服务可用性

    2.1   服务可用性计算方式 服务可用性 =（1 - 服务不可用时间 / 服务月度内总时间）x 100%

    2.2   服务可用性 Rainbond Cloud 提供的服务可用性具体为： Rainbond Cloud 服务可用性不低于 99.95%。 假设当月为 30 天，即用户每月 Rainbond Cloud 服务30 天 x 24 小时 x 60 分钟 x 99.95% = 43178.4分钟，即允许存在 43200 - 43178.4 = 21.6 分钟的不可用时间。

    • （1） 如某服务月度没有达到服务可用性标准，您可以在没有达标的相应服务月度结束后的次月的3日后（每月1-3日为好雨内部结算日）， 通过您相应账户的工单系统或联系服务专员提出赔偿申请。 您提出赔偿申请后好雨会进行相应核实，对于服务月度的服务可用性的计算，若双方出现争议的，双方均同意最终以 Rainbond Cloud 的后台记录为准。
